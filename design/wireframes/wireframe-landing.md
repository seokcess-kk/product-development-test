# Wireframe: Landing Page

## Overview

- **화면**: 랜딩 페이지
- **경로**: `/`
- **목적**: 서비스 가치 제안 및 회원가입 유도
- **진입점**: 직접 URL 입력, 검색, 광고
- **이탈점**: `/signup`, `/login`, `/dashboard` (로그인 상태)

## Elements

| 요소 | 타입 | 설명 | 인터랙션 |
|------|------|------|----------|
| Logo | Image | 서비스 로고 | 클릭 시 홈으로 (현재 페이지 새로고침) |
| 로그인 | Button (Ghost) | 기존 사용자용 | `/login` 이동 |
| 시작하기 | Button (Secondary) | 헤더 CTA | `/signup` 이동 |
| 무료로 시작하기 | Button (Primary, Large) | 메인 CTA | `/signup` 이동 |
| Hero Illustration | Image/Animation | 서비스 느낌 전달 | 없음 (시각적 요소) |
| Feature Card | Card | 주요 기능 소개 | Hover 시 미세 확대 |
| Step Illustration | Image | 사용 방법 설명 | 없음 |
| 후기 Carousel | Carousel | 사용자 후기 | 좌우 스와이프/버튼 |
| Footer Links | Link | 약관, 정보 | 각 페이지 이동 |

## States

### Default (비로그인 상태)
- 위 와이어프레임 그대로 표시
- 헤더: 로그인, 시작하기 버튼 표시

### Logged In (로그인 상태)
- 랜딩 페이지 접근 시 `/dashboard`로 자동 리다이렉트
- 또는 헤더에 "대시보드로 이동" 버튼 표시

### Loading
- 최초 로딩 시 스켈레톤 UI 표시 (Hero, Feature Cards)

### Mobile Scroll
- 헤더 고정 (sticky)
- 스크롤 시 헤더에 그림자 추가

---

## Responsive Behavior

| Breakpoint | 레이아웃 변화 |
|------------|--------------|
| < 640px | 세로 스택, 전체 너비 버튼, 후기 캐러셀 |
| >= 640px | Feature Cards 2열 |
| >= 1024px | Hero 2열 (텍스트 + 이미지), Feature Cards 3열, 후기 2열 |
| >= 1280px | 최대 너비 1200px, 중앙 정렬 |

---

## Annotations

1. **Hero Section**
   - 모바일: 이미지 -> 텍스트 -> CTA 순서 (세로)
   - 데스크톱: 텍스트(좌) + 이미지(우) 2열 구조
   - CTA 버튼: height 48px 이상, 충분한 터치 영역

2. **Feature Cards**
   - 아이콘: 48px, primary-500 색상
   - 제목: text-xl (20px), font-semibold
   - 설명: text-base (16px), gray-600
   - 카드 간격: gap-6 (24px)

3. **How It Works**
   - 3단계 시각화
   - 번호 강조 (primary 색상 원형 배지)
   - 일러스트레이션은 단순화된 UI 모습

4. **Testimonials**
   - 모바일: 캐러셀
   - 데스크톱: 2-3개 동시 표시
   - 인용문 + 이름 + 학년

5. **Final CTA**
   - 배경색 변경 (primary-50 또는 gradient)
   - 버튼 크기 크게 (lg)

6. **Footer**
   - 최소화된 정보
   - 링크: 서비스 소개, 약관, 개인정보, 고객센터

- [x] **일관성과 표준**: 일반적인 랜딩 페이지 패턴
- [x] **미니멀 디자인**: 핵심 정보만, 불필요한 요소 제거

- [x] CTA 명확 ("무료로 시작하기")
