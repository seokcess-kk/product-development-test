# Wireframe: Dashboard

## Overview

- **화면**: 대시보드 (메인 화면)
- **경로**: `/dashboard`
- **목적**: 오늘 학습 현황 확인 및 빠른 액션 수행
- **진입점**: 로그인 성공, 하단 네비게이션, 사이드 메뉴
- **이탈점**: `/plan/new`, `/calendar`, `/records`, `/settings`

| 요소 | 타입 | 설명 | 인터랙션 |
|------|------|------|----------|
| 인사말 | Text | 사용자 이름 포함 | 없음 |

## Responsive Behavior

| Breakpoint | 레이아웃 변화 |
|------------|--------------|
| < 640px | 세로 스택, 하단 네비게이션 |
| >= 640px | 카드 2열 그리드 |
| >= 1024px | 사이드바 + 메인 콘텐츠, 상단 네비게이션 |
| >= 1280px | 사이드바 고정 (280px), 콘텐츠 영역 확장 |

## Annotations

1. **Header**
   - 모바일: Logo + 타이틀 + 알림 + 프로필
   - 데스크톱: Logo + 메뉴 탭 + 알림 + 프로필 드롭다운

2. **인사말**
   - 시간대별 인사 (아침/점심/저녁/밤)
   - 사용자 닉네임 표시

3. **스트릭 카드**
   - 불꽃 아이콘: 24px, warning-500 (활성) / gray-400 (비활성)
   - 숫자 강조: text-2xl, font-bold

4. **진행률 섹션**
   - Progress Bar: height 8px, rounded-full
   - 완료율: text-4xl, font-bold, primary-500
   - 학습 시간: text-sm, gray-500

5. **할 일 리스트**
   - 항목 간격: gap-3 (12px)
   - 과목별 색상 배지 (design-system 참조)
   - 현재 시간대 항목 강조

6. **차트**
   - Bar Chart: 7일 데이터, 최대 높이 100px
   - Pie Chart: 4-5개 섹션, 과목별 색상

7. **하단 네비게이션 (모바일)**
   - 5개 탭: 홈, 캘린더, + (계획), 기록, 설정
   - + 버튼: 강조 (primary 색상, 큰 사이즈)
   - 활성 탭: primary-500, 나머지: gray-400

## Usability Checklist

### Nielsen Heuristics
- [x] **시스템 상태 가시성**: 진행률, 스트릭, D-Day 표시
- [x] **실제 세계와 일치**: "오늘 할 일", "공부 시작하기" 등 친숙한 표현
- [x] **사용자 제어와 자유**: 완료 체크 해제, 건너뛰기 옵션
- [x] **일관성**: 카드, 버튼, 색상 일관됨
- [x] **기억보다 인식**: 현재 할 일 강조 표시

### UX Laws
- [x] **3클릭 규칙**: 주요 액션 2클릭 이내
- [x] **피츠의 법칙**: CTA 버튼 크고 접근 쉬운 위치
- [x] **밀러의 법칙**: 정보 그룹화 (진행률, 할 일, 통계)

### Motivation Design
- [x] 스트릭으로 연속 학습 동기 부여
- [x] 진행률 바로 성취감 시각화
- [x] 완료 시 시각적 피드백 (체크 애니메이션)

### Accessibility
- [x] 색상만으로 정보 전달하지 않음 (아이콘 + 텍스트)
- [x] 충분한 터치 타겟 (44px+)
- [x] 스크린 리더 지원 (aria-label)

## Navigation

- **알림 아이콘**: 알림 리스트 (드롭다운 또는 페이지)
- **프로필 아이콘**: 프로필 메뉴 (드롭다운)
- **일정 항목 클릭**: 일정 상세 모달
- **시작하기 클릭**: `/records` (타이머 시작)
- **더보기 클릭**: `/calendar` (오늘 일정 상세)
- **새 계획 만들기**: `/plan/new`
- **공부 시작**: `/records` (타이머 페이지)
- **하단 네비게이션**:
  - 홈: `/dashboard` (현재)
  - 캘린더: `/calendar`
  - +: `/plan/new`
  - 기록: `/records`
  - 설정: `/settings`
