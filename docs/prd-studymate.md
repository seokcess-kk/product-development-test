## 1. 개요

### 1.1 한 줄 설명

고등학생이 스스로 공부 계획을 세우고 실행할 수 있도록 돕는 가이드형 학습 플래너 웹 서비스

### 1.2 배경 및 목적

**문제 상황**:
- 고등학생들은 무엇을 공부해야 하는지는 알지만, "어떻게" 계획을 세워야 하는지 모름
- 빈 캘린더나 플래너 앞에서 막막함을 느낌
- 계획을 세워도 현실적이지 않아 실천하지 못하고 포기함
- 시간 관리 능력이 부족하여 비효율적으로 공부함

**해결하고자 하는 것**:
- "계획을 관리하는 것보다 계획을 세워주는 것이 더 중요" - 핵심 인사이트
- 빈 캔버스가 아닌, 가이드된 계획 생성 프로세스 제공
- 학교 시간표 기반의 현실적인 시간 블록 추천
- 과목/시험별 템플릿으로 계획 생성의 부담 감소

**기대 효과**:
- 계획 생성에 대한 진입 장벽 제거
- 현실적인 계획으로 완료율 향상
- 꾸준한 학습 습관 형성

### 1.3 목표

1. 고등학생이 5분 이내에 주간 학습 계획을 생성할 수 있게 한다
2. 세운 계획의 실제 실행률(완료율)을 높인다
3. 학습 기록을 통해 자신의 공부 패턴을 파악할 수 있게 한다
4. 매일 접속하여 사용하는 습관을 형성한다

## 2. 타겟 사용자

### 2.1 주요 사용자

| 구분 | 설명 |
|------|------|
| Who | 대한민국 고등학생 (1-3학년) |
| 연령대 | 16-19세 |
| 특성 | 학업에 대한 의지는 있으나 계획 수립과 시간 관리에 어려움을 겪음 |
| 기기 | 스마트폰, 태블릿, PC (웹 반응형 필수) |
| 기술 수준 | 디지털 네이티브, 앱/웹 사용에 익숙함 |

### 2.2 사용자가 겪는 문제

1. **계획 수립의 어려움**: "뭘 어떻게 계획해야 할지 모르겠어요"
2. **비현실적인 계획**: "계획은 세우는데 항상 밀려요"
3. **시간 관리 부재**: "하루가 어떻게 가는지 몰라요"
4. **동기 부여 부족**: "혼자 공부하니까 의지가 안 생겨요"
5. **학습 현황 파악 어려움**: "내가 얼마나 공부했는지 잘 모르겠어요"

### 2.3 사용자 페르소나

#### 페르소나 1: 민준 (고2, 일반고)

**기본 정보**:
- 나이: 17세
- 학교: 서울 소재 일반고 2학년
- 성적: 중상위권 (3등급대)

**특징**:
- 성적을 올리고 싶은 의지가 강함
- 학원 없이 인강으로 혼자 공부
- 계획을 세우면 처음 며칠은 잘 따르다가 점점 밀림

**행동 패턴**:
- 학교 끝나고 저녁 7시부터 공부 시작
- 주말에는 5-6시간 집에서 공부
- 틈틈이 핸드폰으로 유튜브 시청

**니즈**:
- "현실적으로 지킬 수 있는 계획"
- "내가 얼마나 했는지 눈에 보이는 것"
- "오늘 뭘 해야 하는지 바로 알고 싶음"

**Pain Points**:
- 계획 짜는 데 시간이 너무 오래 걸림
- 계획을 세워도 맞는 건지 확신이 없음
- 밀린 계획을 수정하기 귀찮음

#### 페르소나 2: 서연 (고3, 수험생)

**기본 정보**:
- 나이: 18세
- 학교: 경기도 소재 일반고 3학년
- 성적: 상위권 (2등급대)

**특징**:
- 수능까지 남은 시간 관리가 절실함
- 과목별 시간 배분에 고민이 많음
- 규칙적인 생활 패턴 유지 중

**행동 패턴**:
- 새벽 6시 기상, 밤 12시 취침
- 자습 시간에 집중적으로 공부
- 모의고사 일정에 맞춰 계획 수정

**니즈**:
- "D-Day 기준으로 역산해서 계획을 세우고 싶음"
- "과목별 공부 시간 밸런스를 맞추고 싶음"
- "얼마나 꾸준히 했는지 기록으로 보고 싶음"

**Pain Points**:
- 수능까지 할 양은 많은데 시간 배분이 어려움
- 계획 변경이 잦아서 관리가 힘듦

#### 3.3.1 회원 가입/로그인 (P0)

**설명**: 사용자가 서비스에 가입하고 로그인할 수 있는 기능

**인증 방식**:
1. 이메일/비밀번호 가입 및 로그인
2. Google 소셜 로그인

**요구사항**:

| 항목 | 내용 |
|------|------|
| 이메일 형식 | 유효한 이메일 형식 검증 |
| 비밀번호 | 최소 8자, 영문+숫자 조합 |
| Google 로그인 | OAuth 2.0 기반 |
| 세션 유지 | 30일간 로그인 상태 유지 |
| 비밀번호 찾기 | 이메일 인증 후 재설정 |

**화면**:
- `/signup` - 회원가입 페이지
- `/login` - 로그인 페이지
- `/reset-password` - 비밀번호 재설정

**완료 조건**:
- [ ] 유효한 이메일/비밀번호로 회원가입 성공
- [ ] 잘못된 형식 입력 시 에러 메시지 표시
- [ ] Google 계정으로 원클릭 로그인/가입 성공
- [ ] 로그인 후 대시보드로 리다이렉트
- [ ] 새로고침해도 로그인 상태 유지
- [ ] 로그아웃 기능 동작

#### 3.3.2 가이드형 계획 생성 (P0) - 핵심 차별화

**설명**: 위저드 방식으로 사용자를 단계별로 안내하여 학습 계획을 자동 생성하는 기능. 빈 캘린더에 직접 입력하는 것이 아닌, 질문에 답하면 계획이 생성되는 방식.

**핵심 가치**: "계획을 관리하는 것보다 계획을 세워주는 것이 더 중요"

**위저드 단계**:

**Step 1: 목표 설정**
- 시험 선택 (중간고사, 기말고사, 모의고사, 수능, 기타)
- 목표 날짜 입력 (D-Day)
- (선택) 목표 성적/등급 입력

**Step 2: 과목 선택**
- 공부할 과목 선택 (국어, 수학, 영어, 탐구과목 등)
- 과목별 현재 수준/중요도 설정 (상/중/하)
- 과목별 예상 필요 시간 추천

**Step 3: 가용 시간 입력**
- 학교 시간표 입력 (선택)
- 평일 공부 가능 시간대 설정
- 주말 공부 가능 시간대 설정
- 고정 일정 입력 (학원, 과외 등)

**Step 4: 시간 배분 확인**
- 과목별 권장 시간 비율 표시
- 사용자가 조정 가능
- 총 필요 시간 vs 가용 시간 비교

**Step 5: 일정 자동 생성 및 확인**
- 알고리즘 기반 일정 자동 배치
- 주간 계획 미리보기
- 수정 가능
- "이 계획으로 시작하기" 버튼

**시간 배분 알고리즘 기본 로직**:
```
1. D-Day까지 남은 총 가용 시간 계산
2. 과목별 중요도/현재수준에 따라 가중치 부여
3. 가중치 기반으로 과목별 시간 할당
4. 집중력이 높은 시간대에 어려운 과목 배치
5. 연속 2시간 이상 같은 과목 피하기
6. 쉬는 시간 자동 배치 (50분 공부 + 10분 휴식)
```

**과목별 템플릿 예시**:
- 국어: 지문 독해 → 문법 → 문학
- 수학: 개념 → 기본 문제 → 응용 문제
- 영어: 단어 → 독해 → 듣기

**화면**: `/plan/new`

**완료 조건**:
- [ ] 5단계 위저드가 순서대로 진행됨
- [ ] 각 단계에서 이전/다음 이동 가능
- [ ] 과목 선택 시 기본 과목 리스트 제공
- [ ] 시간 배분 결과가 시각적으로 표시됨
- [ ] 자동 생성된 일정을 미리보기로 확인 가능
- [ ] "시작하기" 클릭 시 캘린더에 일정 등록
- [ ] 전체 과정 5분 이내 완료 가능

#### 3.3.3 일정 관리 (P0)

**설명**: 생성된 학습 계획을 캘린더 형태로 보고 관리하는 기능

**뷰 타입**:
1. **주간 뷰 (기본)**: 이번 주 전체 일정을 한눈에 확인
2. **일간 뷰**: 오늘 할 일 상세 목록

**기능 상세**:

| 기능 | 설명 |
|------|------|
| 일정 표시 | 과목별 색상 구분, 시간 블록 형태 |
| 완료 체크 | 개별 일정 완료/미완료 토글 |
| 일정 수정 | 클릭하여 시간/내용 수정 |
| 일정 삭제 | 개별 일정 삭제 |
| 일정 추가 | 수동으로 새 일정 추가 |
| 드래그앤드롭 | 일정을 드래그하여 시간/날짜 변경 (P1) |
| 주간 이동 | 이전/다음 주로 이동 |

**일정 데이터 구조**:
```
- id: 고유 ID
- title: 일정 제목 (예: "수학 - 미적분 개념")
- subject: 과목명
- date: 날짜
- start_time: 시작 시간
- end_time: 종료 시간
- is_completed: 완료 여부
- actual_duration: 실제 소요 시간 (학습 기록 연동)
- memo: 메모
```

**화면**: `/calendar`

**완료 조건**:
- [ ] 주간 뷰에서 7일간의 일정이 시간 블록으로 표시됨
- [ ] 일간 뷰에서 오늘 일정이 리스트로 표시됨
- [ ] 일정 클릭 시 상세 정보 및 수정 가능
- [ ] 완료 체크 시 시각적 변화 (취소선, 색상 변경 등)
- [ ] 드래그앤드롭으로 일정 이동 가능 (P1)
- [ ] 일정 없는 시간대에 클릭하여 새 일정 추가 가능

#### 3.3.4 학습 기록 (P0)

**설명**: 실제 공부한 시간을 기록하고 과목별로 확인하는 기능

**기능 상세**:

| 기능 | 설명 |
|------|------|
| 타이머 | 시작/일시정지/종료 버튼, 경과 시간 표시 |
| 과목 연동 | 타이머 시작 시 과목 선택 또는 일정과 연동 |
| 자동 기록 | 타이머 종료 시 자동으로 학습 기록 저장 |
| 수동 입력 | 타이머 없이 직접 시간 입력 가능 |
| 과목별 통계 | 과목별 누적 학습 시간 표시 |
| 기록 리스트 | 날짜별 학습 기록 확인 |
| 메모 | 학습 내용 간단히 메모 (P2) |

**타이머 동작**:
- 최소 기록 단위: 5분
- 5분 미만 시 기록 저장 여부 확인
- 브라우저 탭 닫아도 타이머 유지 (localStorage)
- 장시간(2시간 이상) 연속 시 휴식 권장 알림

**학습 기록 데이터 구조**:
```
- id: 고유 ID
- user_id: 사용자 ID
- subject: 과목명
- date: 날짜
- duration: 학습 시간 (분)
- schedule_id: 연동된 일정 ID (선택)
- memo: 메모 (선택)
- created_at: 기록 시간
```

**화면**: `/records`

**완료 조건**:
- [ ] 타이머 시작/일시정지/종료 동작
- [ ] 과목 선택 후 타이머 시작 가능
- [ ] 캘린더 일정에서 바로 타이머 시작 가능
- [ ] 타이머 종료 시 자동으로 기록 저장
- [ ] 과목별 누적 학습 시간 표시
- [ ] 날짜별 학습 기록 리스트 확인 가능
- [ ] 기록 삭제 가능

#### 3.3.5 대시보드 (P0)

**설명**: 메인 화면으로, 오늘 학습 현황과 주요 통계를 한눈에 확인

**표시 정보**:

| 영역 | 내용 |
|------|------|
| 오늘의 할 일 | 오늘 일정 리스트, 완료 체크 가능 |
| 오늘 진행률 | 완료/전체 일정 비율 (프로그레스 바) |
| 주간 학습 시간 | 이번 주 총 학습 시간 (차트) |
| 과목별 시간 | 과목별 학습 시간 비율 (파이 차트) |
| 연속 학습일 | 스트릭 카운트 표시 (P1) |
| 다가오는 시험 | D-Day 카운트다운 (설정된 경우) |

**빠른 액션**:
- "오늘 공부 시작하기" → 타이머 페이지로 이동
- "새 계획 만들기" → 계획 생성 위저드로 이동
- 일정 클릭 → 해당 일정 상세/타이머 시작

### 시나리오 1: 첫 가입 후 계획 생성

**사용자**: 민준 (고2, 처음 가입)
**상황**: 2주 후 중간고사를 앞두고 계획을 세우고 싶음
**목표**: 중간고사 대비 학습 계획 생성

**흐름**:
1. 민준이 StudyMate 랜딩 페이지에 접속
2. "시작하기" 버튼을 클릭하여 회원가입 페이지로 이동
3. Google 계정으로 간편 가입 완료
4. 온보딩 화면에서 "계획 만들기" 선택
5. **Step 1**: 시험 종류로 "중간고사" 선택, 날짜 입력
6. **Step 2**: 국어, 수학, 영어, 한국사 과목 선택, 수학을 "상" 중요도로 설정
7. **Step 3**: 평일 저녁 7-11시, 주말 오전 10시-오후 6시 가용 시간 입력
8. **Step 4**: 자동 계산된 과목별 시간 배분 확인, 수학 비율 약간 조정
9. **Step 5**: 자동 생성된 2주 계획 미리보기 확인
10. "이 계획으로 시작하기" 클릭
11. 대시보드로 이동, 오늘 할 일이 표시됨

**성공 조건**: 5분 이내에 전체 과정 완료, 캘린더에 2주 일정 등록

```

├── created_at

├── id (PK)

├── exam_type
├── target_date
├── created_at

├── id (PK)
├── plan_id (FK)
├── name
├── priority (상/중/하)
├── allocated_hours

Schedules (일정)
├── id (PK)
├── user_id (FK)
├── plan_id (FK, nullable)
├── subject
├── title
├── date
├── start_time
├── end_time
├── is_completed
├── actual_duration

StudyRecords (학습 기록)
├── id (PK)
├── user_id (FK)
├── schedule_id (FK, nullable)
├── subject
├── date
├── duration (분)
├── memo
├── created_at

UserStats (사용자 통계)

├── max_streak

```
